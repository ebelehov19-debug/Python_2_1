# Лабораторная работа №1. Источники задач и контракты

## Цель работы
- Освоить duck typing и контрактное программирование на примере источников задач
- Реализовать подсистему приёма задач в платформе обработки задач

## Постановка задачи
Необходимо реализовать подсистему приёма задач в платформе обработки задач. Задачи могут поступать из различных источников, не связанных наследованием, но обязанных реализовывать единый поведенческий контракт.

**Задача** — структура данных, содержащая:
- `id` — уникальный идентификатор задачи
- `payload` — произвольные данные задачи

## Технические требования
- Использование `Protocol` и `runtime_checkable`
- Осмысленное применение `isinstance()` и/или `issubclass()`
- Аннотации типов для всех публичных интерфейсов
- Поддержка расширяемости через реестр плагинов

## Чему я научился
- Применению `Protocol` для определения поведенческих контрактов
- Использованию `@runtime_checkable` для runtime-проверок
- Реализации плагинной архитектуры через реестр фабрик
- Работе с итераторами и генераторами для ленивой загрузки данных
- Созданию CLI-приложений с помощью библиотеки Typer
- Разделению кода на модули по принципу единственной ответственности

## Поддерживаемые источники задач
### 1 Стандартный поток ввода (stdin)
- Формат: id:payload
- Пример: 1:Купить молоко
- Команда: --stdin
### 2 JSON Lines файлы 
- Формат: {"id": "...", "payload": ...}
- Пример: {"id": "1", "payload": "Купить молоко"}
- Команда: --jsonl tasks.jsonl

## Инструкции по запуску

## Примеры использования программы
1. **Чтение из JSONL(JOSN) файла**
```python -m src resd --jsonl source/tasks.jsonl```
2. **Чтение из stdin**
```echo "1: Buy milk" | python -m src read --stdin```
- Многострочный ввод в stdin  
``` 
echo "1:Buy milk
2:Work matan
3:work work" | python -m src read --stdin
``` 
3. **Фильтрация задач**
```python -m src read --jsonl source/tasks.jsonl --contains "Dead Blonde"```
